## 1. 新技能探索：canvas (画布控制)

### 技能简介
**canvas** - 在连接的 OpenClaw 节点（Mac app, iOS, Android）上显示 HTML 内容的工具

### 核心功能
| 能力 | 说明 |
|------|------|
| present | 显示画布内容 |
| hide | 隐藏画布 |
| navigate | 导航到新 URL |
| eval | 在画布中执行 JavaScript |
| snapshot | 截取画布截图 |

### 架构
```
Canvas Host (HTTP:18793) → Node Bridge (TCP:18790) → Node Apps (Mac/iOS/Android)
```

### Tailscale 集成
| Bind 模式 | 服务器绑定 | Canvas URL |
|-----------|------------|------------|
| loopback | 127.0.0.1 | localhost |
| lan | LAN | LAN IP |
| tailnet | Tailscale | Tailscale 主机名 |

### 使用流程
1. 创建 HTML 内容放在 ~/clawd/canvas/
2. 确认 gateway bind 模式
3. 列出连接节点
4. present 内容到节点
5. 可选：navigate/snapshot/hide

### Tailscale 关键洞察
> **为什么 localhost 不工作？**
> - 节点从 bridge 接收 URL
> - 当绑定到 Tailscale 时，节点收到 `http://<tailscale-hostname>:18793/__openclaw__/canvas/<file>.html`
> - 本地 URL 无法在远程节点上访问！

### 技能简介
**apple-reminders** - 通过 remindctl CLI 管理 Apple 提醒事项的工具，支持列表管理、日期筛选、JSON/纯文本输出

### 核心功能
| 能力 | 说明 |
|------|------|
| 查看提醒 | today/tomorrow/week/overdue/all/指定日期 |
| 列表管理 | 列出/创建/删除提醒列表 |
| 创建提醒 | 添加新提醒到指定列表和日期 |
| 完成/删除 | 标记完成或删除提醒 |
| 输出格式 | JSON/TSV/纯文本/静默计数 |

### 安装依赖
```bash
brew install steipete/tap/remindctl
```

### 使用示例
```bash
# 查看提醒
remindctl                    # 今日提醒
remindctl today              # 今天
remindctl tomorrow           # 明天
remindctl week               # 本周
remindctl overdue            # 已过期

# 列表操作
remindctl list               # 列出所有列表
remindctl list Work          # 查看指定列表
remindctl list Projects --create    # 创建列表

# 创建提醒
remindctl add "Buy milk"
remindctl add --title "Call mom" --list Personal --due tomorrow
remindctl add --title "Meeting prep" --due "2026-02-15 09:00"

# 完成/删除
remindctl complete 1 2 3     # 按ID完成
remindctl delete 4A83 --force  # 按ID删除

# 输出格式
remindctl today --json       # JSON格式
remindctl today --plain      # TSV格式
remindctl today --quiet      # 仅计数
```

### 重要区分
> **Apple Reminders vs Cron 提醒**
> - Apple Reminders: 同步到 iPhone/iPad 的提醒事项
> - Cron 提醒: 在 OpenClaw 中发送通知

用户说"提醒我"时需要确认：

- Clawdbot 提醒 → 使用 cron 工具

| 块添加 | 向页面添加块元素 |

### 环境配置
```bash
mkdir -p ~/.config/notion
echo "ntn_your_key_here" > ~/.config/notion/api_key
```

### API 关键参数
- **Notion-Version**: 必须使用 `2025-09-03`（最新版本）
- **数据库 → 数据源**: 新版本中 databases 改名为 data sources
- **两个 ID**: 每个数据库有 database_id 和 data_source_id

### 常用操作示例
```bash
# 搜索页面
curl -X POST "https://api.notion.com/v1/search" \
  -H "Authorization: Bearer $NOTION_KEY" \
  -H "Notion-Version: 2025-09-03" \
  -d '{"query": "page title"}'

# 创建数据库页面
curl -X POST "https://api.notion.com/v1/pages" \
  -H "Notion-Version: 2025-09-03" \
  -d '{
    "parent": {"database_id": "xxx"},
    "properties": {
      "Name": {"title": [{"text": {"content": "New Item"}}]},
      "Status": {"select": {"name": "Todo"}}
    }
  }'

# 查询数据库
curl -X POST "https://api.notion.com/v1/data_sources/{id}/query"
```

### 对我的价值
- **笔记管理** - 读取和写入 Notion 页面
