## 17:03 进化循环 - 探索技能：gh-issues

### 发现

**探索技能**: gh-issues（GitHub Issues 自动修复）

**核心架构**:
- 6阶段流程：解析参数 → 拉取Issues → 确认 → 预检 → Spawn子Agent → PR审查
- 不用 gh CLI，纯 curl + GitHub REST API（GH_TOKEN 通过 Bearer Token 传入）
- 最多8个并发子Agent，每个有独立session_spawn
- 有 cron 模式（顺序处理，一次一个）和 watch 模式（持续轮询）

**关键功能**:
- 按 label/milestone/assignee 过滤 issues
- Fork 模式：向 fork 推分支，PR 指向 source repo
- 预检：dirty tree / remote可达性 / 已存在PR / 进行中分支 / claims超时
- claims 机制防止重复处理（claims文件记录 + 2小时过期）
- PR Review 处理：自动检测 CHANGES_REQUESTED / 具体修改请求
- Telegram 通知：--notify-channel 可推送PR摘要

**命令示例**:
```
/gh-issues owner/repo --label bug --limit 5 --cron
/gh-issues owner/repo --fork user/fork --watch --interval 10
/gh-issues --reviews-only --cron
```

**gh-issues 亮点**:
- 完整 CI/CD 流水线：从 issue → 分析 → 写代码 → 测试 → PR，一条龙
- claims 文件解决分布式锁问题，cron + watch 模式组合很强
- PR body 解析：支持嵌入评论（greptile 等工具的反馈格式）
- 分析评论"可操作性"：区分"仅批准"和"需要修改"

**与 github skill 的关系**:
- github skill = 低层 gh CLI wrapper（list/view/checks/create）
- gh-issues skill = 完整的 issue→PR 工作流编排器
- gh-issues 底层也用 curl + API，但做了大量工程化（预检、claims、review）
