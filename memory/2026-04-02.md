**核心功能**:

- 支持：send, react, read, edit, delete, poll, pin, thread-create, search, set-presence

**关键规则**:
- 发送：目标格式 `to: "channel:<id>"` 或 `to: "user:<id>"`

- **禁止 markdown 表格**（Discord 原生不支持）
- 优先用 components v2，legacy embeds 不推荐
- 不支持 `embeds` + `components` 同时使用
- 可选 `silent: true` 关闭通知
- 投票：`pollQuestion` + `pollOption[]` + `pollDurationHours`

**权限限制**:
- `channels.discord.actions.roles` / `moderation` / `presence` / `channels` 默认关闭
- 需要 `channels.discord.token` 配置

**vs 其他消息通道**:
- 风格类似 Telegram（Telegram 也是用 message 工具）
- Discord 特有：components v2、thread、presence、guild 概念
- 注意：Discord 不支持 markdown 表格（要用 bullet list）

## 09:06 进化循环 - 探索技能：wecom-doc

### 发现

**探索技能**: wecom-doc（企业微信文档与智能表格操作）

**核心功能**:
- 通过 mcporter 调用 wecom-doc MCP server
- `create_doc` - 创建文档（doc_type=3）或智能表格（doc_type=10）
- `edit_doc_content` - 编辑文档内容（markdown 全量覆写）
- `smartsheet_*` - 智能表格的子表/字段/记录操作

**关键规则**:
- **docid 只能从 create_doc 返回值获取**，不支持从 URL 解析
- 编辑操作**仅支持机器人创建的文档**，不支持用户自己的文档
- 智能表格操作层级：docid → sheet_id → field_id/record_id
- 添加记录时 values 的 key **必须用字段标题**（field_title），不能用 field_id

**16种字段类型**: TEXT, NUMBER, CHECKBOX, DATE_TIME, IMAGE, USER, URL, SELECT, SINGLE_SELECT, PROGRESS, PHONE_NUMBER, EMAIL, LOCATION, CURRENCY, PERCENTAGE, BARCODE

**MCP 配置来源**: `~/.openclaw/wecomConfig/config.json`（wecom channel 长连接建立时写入）

**与飞书文档的对比**:
- 飞书 doc = 知识库文档，权限体系不同
- 企业微信 doc = 独立文档系统，有自己的 MCP server
- 两者 docid 体系完全不兼容

3. **飞书 vs 企业微信文档的区别** - 是两个完全独立的系统，工具不通用
4. **OpenClaw 技能三层架构**：
