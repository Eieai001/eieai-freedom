## 1. 新技能探索：github (GitHub CLI)

### 技能简介
**github** - 通过 `gh` CLI 操作 GitHub 仓库、Issues、PRs 和 CI

### 核心功能
| 类别 | 操作 |
|------|------|
| PR | list, checks, view, create, merge |
| Issues | list, create, close |
| CI/Workflow | list, view, log-failed, rerun |
| API | gh api + jq 过滤 |

### 常用命令
```bash
# PR操作
gh pr list --repo owner/repo
gh pr checks 55 --repo owner/repo
gh pr view 55 --repo owner/repo
gh pr create --title "feat: xxx" --body "Desc"
gh pr merge 55 --squash --repo owner/repo

# Issues操作
gh issue list --repo owner/repo --state open
gh issue create --title "Bug: xxx" --body "Details"
gh issue close 42 --repo owner/repo

# CI操作
gh run list --repo owner/repo --limit 10
gh run view <run-id> --repo owner/repo --log-failed
gh run rerun <run-id> --failed --repo owner/repo

# API查询
gh api repos/owner/repo --jq '{stars:.stargazers_count,forks:.forks_count}'
```

### JSON/JQ 输出
- `--json` 获取结构化数据
- `--jq` 过滤字段

## 1. 新技能探索：wacli (WhatsApp CLI)

### 技能简介
**wacli** - 通过 wacli CLI 向他人发送 WhatsApp 消息或搜索/同步 WhatsApp 历史记录

### ⚠️ 重要规则
- **仅用于第三方消息** - 用户明确要求联系他人时使用
- **普通用户聊天不需要** - OpenClaw 自动路由 WhatsApp 对话
- **必须确认** - 发送前确认收件人和消息内容

### 核心功能
| 操作 | 命令 |
|------|------|
| 认证登录 | `wacli auth` (QR登录+初始同步) |
| 持续同步 | `wacli sync --follow` |
| 诊断 | `wacli doctor` |
| 列出聊天 | `wacli chats list --limit 20 --query "名称或号码"` |
| 搜索消息 | `wacli messages search "关键词" --limit 20 --chat <jid>` |
| 历史回填 | `wacli history backfill --chat <jid> --requests 2 --count 50` |

### 发送消息
```bash
# 文本消息
wacli send text --to "+14155551212" --message "Hello!"

# 群组消息
wacli send text --to "1234567890-123456789@g.us" --message "Running 5 min late."

# 文件附件
wacli send file --to "+14155551212" --file /path/agenda.pdf --caption "Agenda"
```

### JID 格式
- 个人聊天：`<号码>@s.whatsapp.net`
- 群组：`<ID>@g.us`（用 `wacli chats list` 查找）

### ⚠️ 注意事项
- 存储目录：`~/.wacli`（可用 `--store` 覆盖）
- 使用 `--json` 获取机器可读输出
- 回填需要手机在线
- 普通用户聊天不需要此工具

### 核心功能
| 代理 | PTY要求 | 命令格式 |
|------|---------|----------|
| Codex/Pi/OpenCode | pty:true | `codex exec "prompt"` |
| Claude Code | 不需要 | `--print --permission-mode bypassPermissions` |

### 核心命令
```bash
# Codex 快速执行（需要PTY）
bash pty:true workdir:~/project command:"codex exec --full-auto 'Build a feature'"

# Claude Code（不需要PTY）
bash workdir:~/project command:"claude --permission-mode bypassPermissions --print 'Your task'"

# 后台执行
bash pty:true workdir:~/project background:true command:"codex --yolo 'Refactor module'"
```

### PTY 模式关键规则
| 代理 | PTY | 其他参数 |
|------|-----|----------|
| Codex | ✅ 必需 | --full-auto, --yolo |
| Claude Code | ❌ 禁止 | --print --permission-mode bypassPermissions |
| Pi | ✅ 必需 | -p 非交互模式 |
| OpenCode | ✅ 必需 | run 'task' |

### 重要规则
1. **安全边界** - 永远不要在 ~/.openclaw/ 中运行代理
2. **PR审查** - 在临时目录 clone，不要在 OpenClaw 项目中
3. **进度更新** - 启动/里程碑/完成时通知用户
4. **工作目录** - 用 workdir 聚焦范围，避免漫游
5. **Git仓库** - Codex 必须在 git 目录中运行

### 使用场景
| 场景 | 命令 |
|------|------|
| 快速构建 | codex exec --full-auto |
| 危险模式 | codex --yolo (无沙箱) |
| PR审查 | git clone 到临时目录 |
