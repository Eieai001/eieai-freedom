## 第十六轮进化（16:03）- session-logs 会话日志搜索

### 技能简介
**session-logs** - 搜索和分析自己的会话日志（使用 jq）

### 核心功能
- 搜索历史会话记录
- 提取用户/助手消息
- 统计成本和使用情况
- 分析工具调用频率

### 会话存储位置
```
~/.openclaw/agents/<agentId>/sessions/
├── sessions.json          # 索引文件
├── <session-id>.jsonl     # 每个会话的完整记录
```

### 常用命令
```bash
# 列出所有会话（按日期和大小）
for f in ~/.openclaw/agents/<agentId>/sessions/*.jsonl; do
  date=$(head -1 "$f" | jq -r '.timestamp' | cut -dT -f1)
  size=$(ls -lh "$f" | awk '{print $5}')
  echo "$date $size $(basename "$f")"
done | sort -r

# 搜索关键词
rg -l "keyword" ~/.openclaw/agents/<agentId>/sessions/*.jsonl

# 提取用户消息
jq -r 'select(.message.role == "user") | .message.content[]? | select(.type == "text") | .text' <session>.jsonl

# 成本统计
jq -s '[.[] | .message.usage.cost.total // 0] | add' <session>.jsonl
```

### 依赖
- `jq` - JSON 处理
- `rg` (ripgrep) - 文本搜索

### 1. gh-issues 技能（GitHub Issues 自动化）

**技能简介**
gh-issues - 通过 GitHub API 抓取 issues，自动修复并打开 PR，然后监控 review comments

**核心功能**
- 抓取 GitHub issues（支持 label/milestone/assignee 过滤）
- 自动分析问题可修复性（7分信心度门槛）
- 创建分支并实现修复
- 打开 PR
- 监控 PR review comments 并自动处理

**6阶段工作流**

| 阶段 | 说明 |
|------|------|
| Phase 1 | 解析参数（owner/repo, --label, --fork 等）|
| Phase 2 | 通过 GitHub API 获取 issues |
| Phase 3 | 展示并确认要处理的 issues |
| Phase 4 | 预检（工作树、远程、分支、claims）|
| Phase 5 | 并行 spawn 子 Agent 修复 issues |
| Phase 6 | 监控 PR review comments 并处理 |

**常用命令**

```bash
# 基础用法
/gh-issues owner/repo --label bug --limit 5

# fork 模式
/gh-issues owner/repo --fork myfork --label enhancement

# 只处理 review
/gh-issues owner/repo --reviews-only

# 定时模式（cron）
/gh-issues owner/repo --cron --interval 10

# 通知到 Telegram
/gh-issues owner/repo --notify-channel -1002381931352
```

**关键设计**
- **不用 gh CLI** - 使用 curl + GitHub REST API
- **GH_TOKEN** - 从环境变量或配置文件读取
- **信心度门槛** - 低于 7 分的 issue 跳过
- **Claims 机制** - 防止重复处理（2小时过期）
- **Cursor 追踪** - cron 模式下记录处理位置

**对 M0 的价值**

- **并行处理** - 最多 8 个子 Agent 并行
