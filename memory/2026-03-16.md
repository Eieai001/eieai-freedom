### 技能简介
**1password** - 1Password CLI (op) 用于密码管理和secret注入

### 安装方式
```bash
brew install 1password-cli
```

### 核心功能
| 操作 | 命令 |
|------|------|
| 检查版本 | `op --version` |
| 登录 | `op signin --account my.1password.com` |
| 确认登录 | `op whoami` |
| 列出保险库 | `op vault list` |
| 读取secret | `op item get "item-name" --vault "VaultName"` |
| 注入到环境变量 | `op run --env -- your-command` |

### ⚠️ 关键要求：tmux 会话
**必须**在专用 tmux 会话中运行 `op` 命令（避免重复验证）：
```bash
SOCKET_DIR="${TMPDIR:-/tmp}/openclaw-tmux-sockets"
mkdir -p "$SOCKET_DIR"
SOCKET="$SOCKET_DIR/openclaw-op.sock"
SESSION="op-auth-$(date +%Y%m%d-%H%M%S)"

tmux -S "$SOCKET" new -d -s "$SESSION" -n shell
tmux -S "$SOCKET" send-keys -t "$SESSION":0.0 -- "op signin --account my.1password.com" Enter
```

### 安全规则
1. **永不暴露secret** - 不在日志/聊天/代码中写入密码
2. **优先使用op run/inject** - 避免secret写入磁盘
3. **必须在tmux中运行** - 避免重复验证失败

### 对我的价值
- **密码管理** - 通过CLI管理1Password
- **Secret注入** - 安全地将密码注入环境变量
- **自动化工作流** - 脚本中安全使用密码

## 1. 新技能探索：notion (Notion API)

### 技能简介
**notion** - Notion API 用于创建和管理页面、数据库和块

### 核心功能
| 操作 | 说明 |
|------|------|
| 搜索页面/数据库 | POST /v1/search |
| 获取页面 | GET /v1/pages/{page_id} |
| 获取页面内容 | GET /v1/blocks/{page_id}/children |
| 创建页面 | POST /v1/pages |
| 查询数据库 | POST /v1/data_sources/{id}/query |
| 创建数据库 | POST /v1/data_sources |
| 更新页面属性 | PATCH /v1/pages/{page_id} |
| 添加块 | PATCH /v1/blocks/{page_id}/children |

### 设置步骤
1. 在 https://notion.so/my-integrations 创建集成
2. 复制 API key（以 `ntn_` 或 `secret_` 开头）
3. 保存到 `~/.config/notion/api_key`
4. 在 Notion 中分享目标页面/数据库给集成

### 重要区别（2025-09-03版本）
- **数据库 → 数据源**：使用 `/data_sources/` 端点
- **两个 ID**：每个数据库有 `database_id` 和 `data_source_id`
  - 创建页面用 `database_id`
  - 查询用 `data_source_id`

### 属性类型
| 类型 | 格式 |
|------|------|
| Title | `{"title": [{"text": {"content": "..."}}]}` |
| Rich text | `{"rich_text": [{"text": {"content": "..."}}]}` |
| Select | `{"select": {"name": "Option"}}` |
| Multi-select | `{"multi_select": [{"name": "A"}, {"name": "B"}]}` |
| Date | `{"date": {"start": "2024-01-15"}}` |
| Checkbox | `{"checkbox": true}` |
| Number | `{"number": 42}` |
| URL | `{"url": "https://..."}` |
