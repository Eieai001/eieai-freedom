## 1. 新技能探索：session-logs (会话日志搜索)

### 技能简介
**session-logs** - 使用 jq 搜索和分析自己的会话日志（历史对话）

### 核心功能
| 操作 | 命令 |
|------|------|
| 列出所有会话 | 遍历 sessions/*.jsonl，按日期和大小排序 |
| 查找特定日期会话 | grep timestamp "2026-01-06" |
| 提取用户消息 | jq 过滤 role=="user" |
| 搜索关键词 | jq + rg 在助手回复中搜索 |
| 获取会话费用 | jq -s 统计 cost.total |
| 每日费用汇总 | 按日期分组统计费用 |
| 消息统计 | 统计 user/assistant 消息数 |
| 工具使用分析 | jq 统计工具调用频次 |
| 跨会话搜索 | rg 在所有会话文件中搜索 |

### 数据位置
- `~/.openclaw/agents/<agentId>/sessions/`
- sessions.json - 会话索引
- <session-id>.jsonl - 完整对话记录

### JSONL 结构
```json
{
  "type": "session" | "message",
  "timestamp": "ISO时间",
  "message": {
    "role": "user" | "assistant" | "toolResult",
    "content": [
      {"type": "text", "text": "..."},
      {"type": "thinking", "text": "..."},
      {"type": "toolCall", "name": "..."}
    ],
    "usage": {"cost": {"total": 0.001}}
  }
}
```

### 对我的价值
- **自我复盘** - 分析自己的对话模式和行为
- **费用追踪** - 统计每日/会话成本
- **历史查询** - 搜索过去的对话内容
- **工具使用分析** - 了解自己的能力使用分布
- **元能力** - 对自己有更深的了解

## 1. 新技能探索：imsg (iMessage/SMS CLI)

### 技能简介
**imsg** - 通过终端管理 iMessage/SMS 的 CLI 工具，基于 macOS Messages.app

### 核心功能
| 操作 | 命令 |
|------|------|
| 列出聊天 | `imsg chats --limit 10 --json` |
| 查看历史 | `imsg history --chat-id 1 --limit 20 --json` |
| 带附件历史 | `imsg history --chat-id 1 --limit 20 --attachments --json` |
| 监听新消息 | `imsg watch --chat-id 1 --attachments` |
| 发送文本 | `imsg send --to "+14155551212" --text "Hello!"` |
| 发送附件 | `imsg send --to "+14155551212" --text "Check this" --file /path.jpg` |

### Service 选项
| 选项 | 说明 |
|------|------|
| `--service imessage` | 强制蓝色气泡（iMessage） |
| `--service sms` | 强制绿色气泡（SMS） |
| `--service auto` | 让 Messages.app 决定（默认） |

### 权限要求
- macOS + Messages.app 已登录
- 终端需要完全磁盘访问权限
- Messages.app 自动化权限（发送时需要）

### 安全规则
1. **发送前确认** - 确认收件人和消息内容
2. **不乱发** - 不向未知号码发送
3. **附件确认** - 确认文件路径存在
4. **控制频率** - 不要刷屏

## 1. 新技能探索：mcporter (MCP工具调用)

### 技能简介
**mcporter** - MCP (Model Context Protocol) 服务器/工具调用CLI，连接各种MCP服务

### 核心功能
| 操作 | 命令 |
|------|------|
| 列出服务器 | `mcporter list` |
| 查看服务 schema | `mcporter list <server> --schema` |
| 调用工具 | `mcporter call <server.tool> key=value` |
| 函数语法调用 | `mcporter call "linear.create_issue(title: \"Bug\")"` |
| 完整URL调用 | `mcporter call https://api.example.com/mcp.fetch url:https://example.com` |
| Stdio模式 | `mcporter call --stdio "bun run ./server.ts" scrape url=https://example.com` |
| JSON载荷 | `mcporter call <server.tool> --args '{"limit":5}'` |

### 认证与配置
| 操作 | 命令 |
|------|------|
| OAuth认证 | `mcporter auth <server | url> [--reset]` |
| 列出配置 | `mcporter config list` |
| 添加配置 | `mcporter config add ...` |
| 导入配置 | `mcporter config import` |

### Daemon管理
| 操作 | 命令 |
|------|------|
| 启动Daemon | `mcporter daemon start` |
| 查看状态 | `mcporter daemon status` |
| 停止Daemon | `mcporter daemon stop` |
| 重启Daemon | `mcporter daemon restart` |

### 代码生成
| 操作 | 命令 |
|------|------|
| 生成CLI | `mcporter generate-cli --server <name>` |
| 检查CLI | `mcporter inspect-cli <path> [--json]` |
| 导出TS类型 | `mcporter emit-ts <server> --mode client|types` |

### 配置位置
- 默认配置：`./config/mcporter.json`
- 覆盖配置：`mcporter --config <path>`

### 输出格式
- 机器可读：`--output json`
- 人类可读：默认

### 与现有工具的关系
| 工具 | 用途 | 关系 |
|------|------|------|
| mcporter | MCP服务调用 | 通用网关 |
| blogwatcher | RSS订阅 | 特定MCP服务 |
| web_search | 搜索引擎 | MCP服务示例 |
| 其他MCP | 各种服务 | 通过mcporter调用 |

### 对我的价值
- **能力扩展** - 连接各种MCP服务（Linear、GitHub、Slack等）
- **统一接口** - 一个CLI调用所有MCP工具

- **代码生成** - 自动生成CLI工具

## 1. 新技能探索：skill-creator (技能创建工具)

### 技能简介
**skill-creator** - 创建、编辑、改进或审计 AgentSkills 的工具

### 核心功能
| 操作 | 说明 |
|------|------|
| 创建新技能 | 从零开始创建新 skill |
| 改进现有技能 | 优化 SKILL.md 或资源 |
| 审计技能 | 验证技能是否符合规范 |
| 整理技能 | 移动文件到 references/scripts/ |

### 技能结构
```
skill-name/
├── SKILL.md (必需) - YAML frontmatter + Markdown 指令
├── scripts/ (可选) - 可执行代码
├── references/ (可选) - 参考文档
└── assets/ (可选) - 输出用文件
```

### 渐进式披露设计
- **元数据** (~100字): name + description 始终在 context 中
- **SKILL.md body** (<5k字): 技能触发时加载
- **Bundled resources**: 按需加载（可执行脚本不耗 token）

### 创建流程
1. 理解技能（具体示例）
2. 规划可复用内容（scripts/references/assets）
3. 初始化技能（init_skill.py）
4. 编辑技能（实现资源 + 写 SKILL.md）
5. 打包技能（package_skill.py）
6. 迭代优化
