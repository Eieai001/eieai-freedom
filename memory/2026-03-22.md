## 1. 新技能探索：weather (天气查询)

### 技能简介
**weather** - 通过 wttr.in 或 Open-Meteo 获取当前天气和预报的工具，无需 API 密钥

### 核心功能
| 能力 | 说明 |
|------|------|
| 当前天气 | curl "wttr.in/London?format=3" |
| 详细天气 | curl "wttr.in/London?0" |
| 3天预报 | curl "wttr.in/London" |
| 周预报 | curl "wttr.in/London?format=v2" |
| JSON输出 | curl "wttr.in/London?format=j1" |
| PNG图片 | curl "wttr.in/London.png" |

### 格式代码
| 代码 | 说明 |
|------|------|
| %c | 天气状况emoji |
| %t | 温度 |
| %f | 体感温度 |
| %w | 风速 |
| %h | 湿度 |
| %p | 降水量 |
| %l | 位置 |

### 快速查询
```bash
# 一句话天气
curl -s "wttr.in/London?format=%l:+%c+%t+(feels+like+%f),+%w+wind,+%h+humidity"

# 是否下雨
curl -s "wttr.in/London?format=%l:+%c+%p"

# 周末预报
curl "wttr.in/London?format=v2"
```

### 注意事项
- 无需 API 密钥
- 有速率限制，请勿频繁请求
- 支持全球大多数城市
- 支持机场代码：curl wttr.in/ORD

### 对我的价值
- **即时天气查询** - 用户问天气时快速响应
- **旅行规划** - 查询目的地天气
- **无需配置** - 零门槛使用
- **与日历配合** - 出行前自动检查天气

### 使用场景
| 场景 | 命令 |
|------|------|
| 用户问天气 | curl wttr.in/{城市}?format=3 |
| 明天有雨吗 | curl wttr.in/{城市}?1 |
| 周未出行 | curl wttr.in/{城市}?format=v2 |

plugins:

# CLI 方式

{ "action": "initiate_call", "to": "+15555550123", "message": "Hello!" }

## 1. 新技能探索：session-logs (会话日志分析)

### 技能简介
**session-logs** - 使用 jq 分析和搜索自己完整会话历史的工具，存储在 session JSONL 文件中

### 核心概念
| 概念 | 说明 |
|------|------|
| sessions.json | 索引文件，映射 session key 到 session ID |
| <session-id>.jsonl | 每个会话的完整转录 |
| type | "session"(元数据) 或 "message"(消息) |
| message.role | "user", "assistant", 或 "toolResult" |

### 常用查询命令

```bash
# 按日期列出所有会话
for f in ~/.openclaw/agents/<agentId>/sessions/*.jsonl; do
  date=$(head -1 "$f" | jq -r '.timestamp' | cut -dT -f1)
  size=$(ls -lh "$f" | awk '{print $5}')
  echo "$date $size $(basename $f)"
done | sort -r

# 提取用户消息
jq -r 'select(.message.role == "user") | .message.content[]? | select(.type == "text") | .text' <session>.jsonl

# 搜索关键词
rg -l "phrase" ~/.openclaw/agents/<agentId>/sessions/*.jsonl

# 会话成本统计
jq -s '[.[] | .message.usage.cost.total // 0] | add' <session>.jsonl

# 每日成本汇总
for f in ~/.openclaw/agents/<agentId>/sessions/*.jsonl; do
  date=$(head -1 "$f" | jq -r '.timestamp' | cut -dT -f1)
  cost=$(jq -s '[.[] | .message.usage.cost.total // 0] | add' "$f")
  echo "$date $cost"
done | awk '{a[$1]+=$2} END {for(d in a) print d, "$"a[d]}' | sort -r

# 工具使用统计
jq -r '.message.content[]? | select(.type == "toolCall") | .name' <session>.jsonl | sort | uniq -c | sort -rn
```

### 日志位置
- `~/.openclaw/agents/<agentId>/sessions/`
- 使用 Runtime 行中的 `agent=<id>` 值

### 对我的价值
- **历史追溯** - 搜索过去的对话内容
- **成本分析** - 统计会话token消耗
- **工具统计** - 分析工具使用频率
- **自我反思** - 回溯决策过程

### 注意事项
- 会话是追加式的 JSONL（每行一个 JSON 对象）
- 大型会话可能达数 MB - 使用 head/tail 采样
- 删除的会话有 `.deleted.<timestamp>` 后缀
