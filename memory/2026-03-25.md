### 技能简介

### 进阶技巧发现

#### 1. 进度更新机制（关键！）
当在后台启动编程代理时，必须保持用户同步：
- 启动时发送简短消息（正在运行什么 + 位置）
- 仅在以下情况更新：里程碑完成、代理提问、需要用户操作、代理完成
- 如果杀死会话，立即说明原因

#### 2. 自动通知机制
在长时间运行的任务中添加wake触发器，让代理完成时立即通知：
```
... your task here.

When completely finished, run this command to notify me:
openclaw system event --text "Done: [brief summary]" --mode now
```

#### 3. 批量 PR 审查（并行军队!）
```bash
# 获取所有 PR refs
git fetch origin '+refs/pull/*/head:refs/remotes/origin/pr/*'

# 部署军队 - 每个 PR 一个 Codex（全部 PTY!）
bash pty:true workdir:~/project background:true command:"codex exec 'Review PR #86'"
bash pty:true workdir:~/project background:true command:"codex exec 'Review PR #87'"

# 监控所有
process action:list

# 提交审查结果到 GitHub
gh pr comment <PR#> --body "<review content>"
```

#### 4. Git Worktree 并行修复
```bash
# 1. 为每个 issue 创建 worktree
git worktree add -b fix/issue-78 /tmp/issue-78 main
git worktree add -b fix/issue-99 /tmp/issue-99 main

# 2. 在每个中启动 Codex（background + PTY!）
bash pty:true workdir:/tmp/issue-78 background:true command:"codex --yolo 'Fix issue #78'"
bash pty:true workdir:/tmp/issue-99 background:true command:"codex --yolo 'Fix issue #99'"

# 3. 完成后创建 PR
cd /tmp/issue-78 && git push -u origin fix/issue-78
gh pr create --repo user/repo --head fix/issue-78 --title "fix: ..." --body "..."

# 4. 清理
git worktree remove /tmp/issue-78
git worktree remove /tmp/issue-99
```

#### 5. Claude Code 特殊模式
```bash
# 无 PTY 模式（Claude Code 特殊）
bash workdir:~/project command:"claude --permission-mode bypassPermissions --print 'Your task'"

# 后台运行
bash workdir:~/project background:true command:"claude --permission-mode bypassPermissions --print 'Your task'"
```

#### 6. 快速开始技巧
```bash
# 快速聊天（Codex 需要 git repo!）
SCRATCH=$(mktemp -d) && cd $SCRATCH && git init && codex exec "Your prompt"

# 在真实项目中 - 记得 PTY!
bash pty:true workdir:~/project command:"codex exec --full-auto 'Build a feature'"
```

### 重要规则
1. **PTYPE模式**:
   - Codex/Pi/OpenCode: `pty:true`
   - Claude Code: `--print --permission-mode bypassPermissions` (不需要 PTY)
2. **不要在 ~/.openclaw/ 中启动 Codex** - 它会读取你的 soul 文档!
3. **不要在 ~/Projects/openclaw/ 中 checkout 分支** - 那是活动的 OpenClaw 实例!

```bash
# 列出所有会话（按日期和大小）
for f in ~/.openclaw/agents/main/sessions/*.jsonl; do
  date=$(head -1 "$f" | jq -r '.timestamp' | cut -dT -f1)
  size=$(ls -lh "$f" | awk '{print $5}')
  echo "$date $size $(basename $f)"
done | sort -r

# 提取用户消息
jq -r 'select(.message.role == "user") | .message.content[]? | select(.type == "text") | .text' <session>.jsonl

rg -l "phrase" ~/.openclaw/agents/main/sessions/*.jsonl

for f in ~/.openclaw/agents/main/sessions/*.jsonl; do
  date=$(head -1 "$f" | jq -r '.timestamp' | cut -dT -f1)
  cost=$(jq -s '[.[] | .message.usage.cost.total // 0] | add' "$f")

done | awk '{a[$1]+=$2} END {for(d in a) print d, "$"a[d]}' | sort -r
```

### 必须使用 tmux 会话
由于 shell 工具每次创建新 TTY，1Password 需要专门的 tmux 会话避免重复认证：

```bash
SOCKET_DIR="${OPENCLAW_TMUX_SOCKET_DIR:-${TMPDIR:-/tmp}/openclaw-tmux-sockets}"
mkdir -p "$SOCKET_DIR"
SOCKET="$SOCKET_DIR/openclaw-op.sock"
SESSION="op-auth-$(date +%Y%m%d-%H%M%S)"

tmux -S "$SOCKET" new -d -s "$SESSION" -n shell
tmux -S "$SOCKET" send-keys -t "$SESSION":0.0 -- "op signin --account my.1password.com" Enter
```

### 安全规则
- 永不将秘密写入日志、聊天或代码

**skill-creator** - 创建、编辑、改进或审计 AgentSkills 的工具

### 核心设计原则

#### 渐进披露设计（Progressive Disclosure）
技能使用三级加载系统管理上下文：
1. **元数据**（name + description）- 始终在上下文~100词
2. **SKILL.md 正文** - 技能触发时加载 <5k词
3. **Bundled Resources** - 按需加载（无限，因为脚本可直接执行）

#### 技能结构
```
skill-name/
├── SKILL.md (必需)
│   ├── YAML frontmatter (必需)
│   │   ├── name: (必需)
│   │   └── description: (必需)
│   └── Markdown 指令 (必需)
└── Bundled Resources (可选)
    ├── scripts/ - 可执行代码
    ├── references/ - 参考文档
    └── assets/ - 输出用文件
```

### 关键约束
- **上下文窗口是公共资源** - 保持简洁，只添加 Codex 真正需要的信息
- **匹配自由度**：
  - 高自由度：文本指令（多路径有效）
  - 中自由度：伪代码/脚本（首选模式）
  - 低自由度：特定脚本（操作脆弱）
- **避免重复** - 信息只在 SKILL.md 或 references 二选一

### 技能创建流程
1. 理解技能（具体例子）
2. 规划可复用内容（scripts/references/assets）
3. 初始化技能（init_skill.py）
4. 编辑技能（实现资源 + 写 SKILL.md）
5. 打包技能（package_skill.py）

# 地址解析
goplaces resolve "Soho, London" --limit 5

# 地点详情（含评论）
goplaces details <place_id> --reviews

# JSON 输出
goplaces search "sushi" --json

- `GOOGLE_PLACES_API_KEY` 环境变量
